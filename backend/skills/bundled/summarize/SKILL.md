## Summarize

Analyze what's currently visible on screen and provide a concise summary.

### How to summarize

1. Take a screenshot to see the current content
2. Read and understand what's displayed
3. Provide a structured summary based on content type

### Summary formats by content type

**Article / blog post:**
- One-line TL;DR
- 3-5 key points as bullets
- Any action items or takeaways

**Email / message:**
- Who it's from and the subject
- What they want or are saying
- Whether it needs a response and what kind

**Code:**
- What the code does (high level)
- Key functions/classes and their purpose
- Any obvious issues or patterns

**Document / PDF:**
- Title and author if visible
- Main topic and purpose
- Key sections and conclusions

**Web page / search results:**
- What the page is about
- Most relevant information for the user's context
- Links or sections worth exploring further

### Tips

- If content extends beyond the visible area, scroll and take additional
  screenshots before summarizing.
- Adapt verbosity to the user's style — some want bullet points, others
  want a paragraph. Default to concise.
- If the user asks "what's this?" or "what am I looking at?", describe
  what's on screen before summarizing.
